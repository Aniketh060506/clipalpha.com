Write a complete AWS Lambda function in Node.js 20 called "unlock" for pastit.site. Environment variables: CONTENT_BUCKET, CLIPS_TABLE, ALLOWED_ORIGIN This function handles POST /api/clip/{slug}/unlock Request body: { passwordHash: string } — OR empty body if clip is not encrypted What it must do: 1. Read DynamoDB record for slug. Return 404 if not found or expired. 2. If wrongAttempts >= 5, return 423 JSON: { error: "locked", message: "This clip is locked due to too many wrong attempts" } 3. If isEncrypted is true: a. Compare submitted passwordHash with stored passwordHash b. If wrong: atomically increment wrongAttempts in DynamoDB using ConditionExpression (wrongAttempts < 5). Return 401 JSON: { error: "wrong_password", attemptsRemaining: 5 - newCount } c. If correct: proceed to step 4 4. If isEncrypted is false: proceed directly to step 4 5. Fetch blob from S3 at record.s3Key 6. Atomically increment viewCount in DynamoDB 7. If burnAfterRead is true: delete DynamoDB record AND delete S3 object 8. Else if maxViews is set and viewCount >= maxViews: delete DynamoDB record and S3 object 9. Return 200 JSON: { blob: <base64 string from S3>, salt: record.salt, iv: record.iv, isEncrypted: record.isEncrypted, contentType: record.contentType, fileName: record.fileName, integrityHash: record.integrityHash } Return proper CORS headers. Use AWS SDK v3. Use bcryptjs (NOT bcrypt native) for password hash comparison. Install bcryptjs by including it in package.json, or use a simple SHA-256 comparison if the frontend sends pre-hashed values — note which approach you use. PASSWORD APPROACH: The frontend will hash the password with SHA-256 before sending. The Lambda stores this hash and compares it. Real bcrypt is not needed because the encryption key uses PBKDF2 — the hash is just a door check, not the encryption key.
